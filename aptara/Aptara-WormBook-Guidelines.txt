WormBook Guideline

This document is from Aptara. I converted it to plaintext so that I could easily edit in my shell.

Note: THIS IS NOW THE REFERENCE DOCUMENT.

1. Convert the Word Document to text format and retain the layout (i.e. bold, italics) etc.

2. Coding will be as per DocBook 4.4CR2 DTD.

3. Code References and Figure legends in separate files.

4. Add footnote to Title as described in Specifications.

5. Date should be formatted as MMM DD, YYYY in  title footnote

6. Code author(s) in <author> tag

7. Author Correspondence should be coded within <author> tag as:
<remark><footnote><para>To whom correspondence should be addressed. Tel. 00000000. E-mail: a.gartner@dundee.ac.uk</para></footnote></remark>

8. Code affiliations in <affiliation> tag.

9. Abstract will be bold.

10. Sections to be converted as sect1, sect2 as per nesting of sections

11. Para as <para>

12. Figures to be converted as: Resolution 300 DPI. Image width 1800 pixels maximum and should be in JPEG format.

13. Figures to be captured in <mediaobject><imageobject>. Image width and alignment will be defined as per display in PDF.

14. Tables to converted as cals table.

15. Colwidth and align defined in colspec according to the table column.

16. Use <link> tag for CrossRef linking

17. For external links use <ulink> tag with role="_blank" attribute.

18. For Numbered list use <orderedlist> with attribute value as numeration="arabic|upperalpha|loweralpha|upperroman|lowerroman"

19. For unnumbered List use <itemizedlist>

20. Simple list converted as <simplelist> <member> …. </member> …. </simplelist>. For multiple columns use attributes COLUMNS and TYPE.

21 .Structure references in cell journal format as follows:
“<author surname>, <author firstname abbreviations without separation>. (YYYY). Article title. Journal title in ISO abbreviations Volume in italics, page numbers.” Use “and” before the last author name.
